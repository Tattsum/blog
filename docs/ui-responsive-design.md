# Zenn 風 UI とレスポンシブ設計

Zenn（[zenn.dev](https://zenn.dev)）のような読みやすく整った見た目と、スマートデバイス（スマートフォン・タブレット）での利用を前提とした設計方針をまとめる。実装は本ドキュメントと [frontend-design.md](frontend-design.md) を前提に進める。

---

## 1. 目的・ゴール

| 項目 | 内容 |
| --- | --- |
| **Zenn 風の見栄え** | 技術ブログ・記事閲覧に適した、余白とタイポグラフィを整えた UI。装飾は控えめにし、コンテンツの可読性を最優先にする。 |
| **スマートデバイス対応** | スマートフォン・タブレットで閲覧・操作しやすいレイアウトとタッチ操作に対応する。 |

---

## 2. Zenn 風 UI の要件

### 2.1 タイポグラフィ

- **フォント**: 本文は既存の `--font-geist-sans` と日本語フォールバック（Hiragino Kaku Gothic ProN, Noto Sans JP）を維持。見出し・本文で一貫したファミリーを使う。
- **行間**: 本文は `line-height: 1.7` 〜 `1.8` を維持。長文でも負担が少ないようにする。
- **見出し階層**: 記事タイトル（h1）は 1.75rem 前後、セクション見出し（h2/h3）は段階的に小さく。`post-body` 内の h2/h3/h4 はマージンとフォントサイズを揃え、縦のリズムを保つ。
- **1 行の長さ**: 本文のカラム幅は最大 720px 程度（既存 `.container`）のまま。約 66 文字前後を目安にし、広すぎる画面では中央寄せで余白を確保する。

### 2.2 余白・スペーシング

- **コンテナ**: デスクトップでは `max-width: 720px`、左右 `padding: 24px` 以上。モバイルでは左右 16px 〜 20px 程度に縮小（[4. レスポンシブ](#4-レスポンシブ) 参照）。
- **記事一覧**: 各記事ブロック間は 24px 〜 32px の余白。区切り線（`border-bottom`）の場合は `padding-bottom` と揃える。
- **記事詳細**: タイトルと日付の間 8px、日付と本文の間 24px。本文中の見出し・段落・リストの前後も 8px 〜 16px で統一する。
- **バーティカルリズム**: 本文の `line-height` を基準に、見出しや段落の `margin` をその倍数（例: 1.5em, 2em）で揃えると読みやすい。

### 2.3 コンポーネントの見た目

- **ヘッダー**: ロゴ（ブログ名）とナビゲーションを 1 行にまとめる。モバイルではハンバーガーまたは縦積みに変更する検討。
- **記事一覧カード**: タイトル・日付・要約を明確に区切り、サムネイルがある場合はカード上部に配置。ホバー時はタイトル色をリンク色に変更（既存方針を維持）。
- **記事詳細**: サムネイル → タイトル → 日付 → 本文の流れ。本文中のコードブロックは `pre` / `code` で背景色・角丸・パディングを適用（既存の `.post-body` を拡張）。
- **ボタン・リンク**: リンクは下線はホバー時のみ、または常に薄い下線。ボタンは角丸 4px 〜 8px、パディングを十分に取りタップしやすくする。

### 2.4 色・コントラスト

- 既存の CSS 変数（`--foreground`, `--muted`, `--link`, `--code-bg` 等）を維持。ダークモードは `prefers-color-scheme: dark` に従う。
- 背景と文字のコントラストは WCAG 2.1 の AA 相当を目安にする（既存テーマで配慮済みを前提に、必要なら調整）。

## 3. スマートデバイス対応の要件

- **ビューポート**: `viewport` メタタグで `width=device-width, initial-scale=1` を指定（Next.js デフォルトで設定される想定）。横向き・縦向きの両方でレイアウトが崩れないようにする。
- **タッチターゲット**: リンク・ボタンは最小 44px × 44px（Apple HIG / Material の推奨）を目安にし、特にナビ・検索ボタン・記事一覧のタイトル周りでタップしやすくする。
- **フォントサイズ**: モバイルで 16px 未満にしない（ズーム無効時でも読みやすいように）。必要に応じて `clamp()` で可変にする。
- **横スクロールの防止**: 本文やコードブロックが画面幅をはみ出さないよう、`overflow-x: auto` と `max-width: 100%` を適用する（コードブロックは既存で対応済み）。

## 4. レスポンシブ

### 4.1 ブレークポイント

| 名前 | 幅の目安 | 用途 |
| --- | --- | --- |
| **mobile** | 〜 599px | スマートフォン縦 |
| **tablet** | 600px 〜 1023px | スマートフォン横・タブレット |
| **desktop** | 1024px 〜 | PC・大画面 |

- CSS では `min-width` のメディアクエリでモバイルファーストにすることが推奨。例: 基本はモバイル用スタイルとし、`@media (min-width: 600px)` でタブレット以上、`@media (min-width: 1024px)` でデスクトップ用を上書きする。

### 4.2 コンテナ・パディング

- **mobile**: コンテナ左右 `padding: 16px` または `20px`。ヘッダーのナビ・検索は縦に並べるか、アイコン化を検討。
- **tablet**: 左右 `padding: 20px` 〜 `24px`。記事一覧は 1 列のままでも可。
- **desktop**: 既存どおり `max-width: 720px`、左右 `padding: 24px`。

### 4.3 ヘッダー・ナビゲーション

- **mobile**: ロゴとナビを 2 行に分ける、またはハンバーガーメニューでナビを開閉する。検索はアイコンタップでフォーム表示にする方式も検討。
- **tablet / desktop**: 1 行レイアウトを維持。ナビリンクと検索フォームの間隔を十分に取る。

### 4.4 記事一覧・カード

- サムネイル画像は `width: 100%`、`max-width` で幅を制限し、アスペクト比を維持（`aspect-ratio` や `height: auto`）。モバイルではカード幅いっぱいに表示してよい。
- タイトル・日付・要約のフォントサイズはモバイルでも 16px 前後を維持。

### 4.5 記事詳細

- サムネイル・タイトル・本文はいずれもコンテナ幅内に収める。本文中の画像・iframe も `max-width: 100%` で overflow を防ぐ。

---

## 5. 実装の優先順序（案）

1. **グローバル・コンテナ**: `globals.css` にメディアクエリを追加し、コンテナのパディングをブレークポイントごとに変更する。
2. **ヘッダー**: モバイルでナビ・検索のレイアウトを変更（縦積み or ハンバーガー）、タッチターゲットを確保する。
3. **記事一覧**: カード間の余白・サムネイルのレスポンシブを整える。
4. **記事詳細**: タイトル・日付・本文の余白と、`post-body` 内の見出し・段落・リストのスタイルを統一する。
5. **管理画面**: 必要に応じてフォームやボタンのタッチターゲット・幅を調整する。
